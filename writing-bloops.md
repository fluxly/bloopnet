# writing bloops

a small cookbook for communicating on bloopnet

---

# what is a bloop

a bloop is a tiny low bandwidth text transmission.

bloops are intentionally constrained.

the goal is not efficiency.
the goal is texture.

bloopnet is like twitter only better because it is worse.

bloops are:

* small
* lowercase
* slightly ambiguous
* emotionally compressed
* radio friendly
* poetic by necessity

a bloop should feel:

* overheard
* transmitted
* weathered
* incomplete
* human

not optimized.

---

# the base alphabet

bloops use a fixed 5 bit alphabet.

## base character table

| value | meaning |
| ----- | ------- |
| 0     | space   |
| 1-26  | a-z     |
| 27    | -       |
| 28    | ?       |
| 29    | |       |
| 30    | return  |
| 31    | esc     |

there are no uppercase letters.

numbers should be written out.

example:

bad:

```text
i have 14 radios
```

good:

```text
i have fourteen radios
```

---

# why lowercase only

lowercase:

* softens tone
* reduces shouting
* removes hierarchy
* increases visual texture
* makes messages feel more equal

all bloops exist on the same visual plane.

---

# why punctuation is limited

limited punctuation slows communication down.

modern social media encourages:

* reaction
* optimization
* precision warfare
* performance

bloops encourage:

* interpretation
* pacing
* implication
* atmosphere

the absence of punctuation is part of the language.

---

# the pipe character

pipes are symmetrical parenthetical expressions.

example:

```text
i saw |a strange bird| near the river
```

pipes can mean:

* aside
* emphasis
* whisper
* uncertainty
* framing
* emotional coloration

they are intentionally flexible.

---

# returns

returns are meaningful.

a return is not just formatting.
it is a structural pause.

example:

```text
weather feels strange tonight
dogs barking near the harbor
```

returns create:

* rhythm
* stanza
* pacing
* silence

they consume space in the bloop budget just like any other symbol.

---

# esc sequences

esc opens extension space.

the canonical form is:

```text
esc bank index
```

this allows future extensions without changing the core language.

possible future banks:

* punctuation
* emotion markers
* protocol symbols
* pictograms
* audio hints
* experimental glyphs

important:
the base language should remain usable without esc.

esc should feel intentional and rare.

---

# the sixty four character bloop

the standard bloop size is:

```text
64 symbols
```

this is important because:

```text
64 x 5 bits = 320 bits = 40 bytes
```

which fits neatly into low bandwidth radio systems like lora.

a bloop should feel:

* complete
* compressed
* intentional

not sprawling.

---

# writing within constraints

writing bloops is closer to:

* field notes
* poetry
* radio dispatch
* overheard conversation
* weather reports
* dreams
* fragments

than blogging.

a good bloop leaves space for the reader.

---

# bloop writing techniques

## implication

say less than you know.

instead of:

```text
i am sad because the town is abandoned and lonely
```

try:

```text
all the windows downtown were dark by six tonight
```

---

## observation over declaration

instead of:

```text
everything feels terrible lately
```

try:

```text
the grocery freezer motors sounded louder than usual
```

---

## use weather

weather creates emotional context very efficiently.

examples:

* fog
* rain
* sodium lights
* harbor wind
* insects
* fluorescent hum
* static
* snow

---

## let ambiguity survive

bloops should not explain everything.

good:

```text
your message arrived damaged
i trusted it more afterward
```

less good:

```text
your message had corruption due to radio interference
```

---

# what bloops are not

bloops are probably not good for:

* discourse
* branding
* metrics
* arguments
* hot takes
* optimization
* influencer behavior

that is intentional.

---

# suggested philosophy

every inconvenience is potentially a cultural affordance.

constraints create style.

slowness creates texture.

ambiguity creates humanity.

small transmissions can still carry emotional weight.

especially small transmissions.

---

# possible future directions

bloops may eventually travel through:

* lora radio
* ble beacons
* serial terminals
* mesh networks
* led signs
* tiny thermal printers
* gallery installations
* ambient sound environments

the protocol should remain:

* tiny
* durable
* understandable
* hackable
* physically representable

a bloop should be able to survive:

* weak signals
* old hardware
* packet loss
* imperfect transmission

and still feel human.

---

# final advice for writing bloops

do not try to sound clever.

do not explain too much.

write like:

* you are near a harbor at night
* the transmission might fail
* somebody unknown may hear it later
* the weather matters
* silence matters too
